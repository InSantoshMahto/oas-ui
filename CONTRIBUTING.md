# Contributing to @insantoshmahto/oas-ui

Thank you for considering contributing to this project — contributions of all kinds are welcome.

What to expect
- Use issues to discuss proposed changes before opening large PRs.
- Keep PRs focused and small; multiple small PRs are easier to review.

How to contribute
1. Fork the repository and create a branch for your change.
2. Develop your change on the branch. Follow the existing code style and keep changes minimal.
3. Run the TypeScript build locally: `npm install && npm run build`.
4. Open a pull request against `main` (or the default branch).

Commit messages
- Use concise messages. For larger changes, the PR description should explain the rationale.

Code style
- This project uses TypeScript with `strict` turned on. Keep types explicit where helpful.

Reporting issues
- Use the issue templates (`.github/ISSUE_TEMPLATE`) to file bugs or feature requests.

License
- By contributing, you agree that your contributions will be licensed under the project's MIT license.
